# 의사결정 기록: Streamlit 대시보드

## DEC-A1: 데이터 저장 포맷 — Parquet
- **맥락**: 100만 건 JSON(4GB)을 매번 로드하면 메모리 초과 및 속도 문제
- **선택지**: JSON 직접 로드 / SQLite / Parquet
- **결정**: Parquet (날짜별 파티셔닝)
- **근거**: pandas 최적화, 컬럼 단위 압축으로 용량 1/5~1/10, 날짜 필터 시 필요한 파일만 로드

## DEC-A2: 웹 프레임워크 — Streamlit
- **맥락**: 졸업 작품으로 데모 가능한 웹 앱 필요
- **결정**: Streamlit
- **근거**: Python만으로 웹 앱 구축, 기존 ML 코드 재활용, 빠른 개발

## DEC-A3: 시각화 라이브러리 — Plotly
- **맥락**: Streamlit 내에서 인터랙티브 차트 필요
- **결정**: Plotly (Streamlit 네이티브 지원)
- **근거**: 호버, 줌 등 인터랙션 지원, st.plotly_chart()로 바로 사용

## DEC-F1: 데이터 변환 파이프라인 분리
- **결정**: convert_to_parquet.py를 별도 스크립트로 분리
- **근거**: 변환과 분석의 관심사 분리

## DEC-F2: 모델 재학습은 on-demand
- **결정**: 사용자가 버튼을 눌러야 재학습
- **근거**: 재학습에 시간이 걸리므로 의도적 트리거가 적절

## DEC-F3: 유저별 예측은 학습된 모델에서 predict
- **결정**: 이미 학습된 모델의 predict_proba()로 확률 반환
- **근거**: 실시간 재학습은 불필요, 기존 모델 재활용
